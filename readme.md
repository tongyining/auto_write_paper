# 📄 AI 论文生成器（团队自用版）

一个完全自包含的 AI 论文生成工作流，无需外部依赖（除 Node.js 和 API Key）。输入论文标题、学位和字数要求，即可自动生成符合学术规范的大纲和全文，支持自定义大纲、在线编辑、一键下载。

---

## ✨ 功能特性

- **一键生成大纲**：基于标题自动生成章节目录（含二级、三级标题）
- **灵活编辑大纲**：支持在线修改章节标题、描述，添加特性标记（表格、图表等）
- **逐章生成全文**：按大纲结构逐章撰写正文，自动合成完整论文
- **多种学位支持**：函授、本科、硕士，不同学位采用不同字数要求和写作深度
- **自定义大纲导入**：支持粘贴 Markdown 格式的大纲，AI 自动解析为结构化 JSON
- **评价反馈**：为已完成任务打分并留下建议，帮助持续改进
- **任务管理**：按用户标识隔离任务，支持管理员查看全部任务
- **访问验证**：内置简单密钥验证，保护服务不被未授权访问

---

## 🛠 技术栈

- Node.js (v14+)
- Express
- 智谱 AI API (GLM-4-Flash/GLM-4-Plus) 或兼容 OpenAI 格式的接口
- 纯前端 HTML + CSS + JavaScript（无需构建工具）

## 📦 安装与运行

### 前提条件

- 安装 [Node.js](https://nodejs.org/)（建议 v16 或以上）
- 获取一个有效的 AI API Key（例如智谱 AI 的 `ZHIPUAI_API_KEY`）

### 1. 克隆仓库

```bash
git clone https://github.com/your-username/your-repo.git
cd your-repo
```

### 2. 安装依赖

在项目根目录下执行：

```bash
npm install
```

### 3. 配置环境变量

创建 `.env` 文件（位于项目根目录），内容如下：

```env
# 访问验证密钥（自定义字符串，用于保护服务）
AUTH_SECRET_KEY=your_secret_key_here
# 管理员密钥（可选，拥有查看所有任务的权限）
ADMIN_SECRET_KEY=your_admin_key_here
```

> **注意**：若未设置 `AUTH_SECRET_KEY`，服务启动时会报错并退出。您可以在 `.env` 中随意填写一串字符，访问时需输入该密钥才能进入系统。

### 4. 启动服务

```bash
node workflows/paper-generation-local/server.js
```

默认监听 `http://0.0.0.0:3000`。启动后，用浏览器访问 `http://localhost:3000` 即可。

## 🚀 使用指南

### 1. 首次访问与验证

打开浏览器进入 `http://localhost:3000`，会弹出验证框，请输入您在 `.env` 中设置的 `AUTH_SECRET_KEY`（或管理员密钥），通过后进入主界面。

### 2. 配置 API Key

点击页面右上角的 **配置⚙️** 按钮，在弹出的模态框中：

- 选择模型（如 智谱 GLM-4-Flash）
- 填入您的 API Key
- 点击 **测试连接**，若返回正确结果（如 `2`），则保存配置

配置会保存在当前目录的 `user_config.json` 中，下次启动无需重复输入。

### 3. 生成论文

1. **输入论文信息**  
   - 标题：必填  
   - 学位：函授 / 本科 / 硕士  
   - 字数：点击预设按钮（如 3000、1万、3万等）  
   - 额外要求：可选，例如“重点突出案例分析”  
   - 投喂资料：可粘贴参考文本或上传 `.txt`/`.md` 文件，AI 将参考这些内容

2. **生成大纲**  
   点击 **生成大纲** 按钮，等待几秒至几分钟，大纲将自动显示在下方编辑器中。

3. **编辑大纲（可选）**  
   - 点击章节旁的 **➕ 子节点** 可添加子章节  
   - 点击 **🗑 删除** 可移除章节  
   - 拖拽上下箭头（⬆⬇）调整顺序  
   - 为三级标题添加特性（表格、图表、代码、公式），AI 生成时会在对应位置插入相应内容

4. **自定义大纲（替代 AI 生成）**  
   您也可以直接粘贴 Markdown 格式的大纲到 **自定义大纲** 文本框，点击 **转换并应用**，系统会解析并替换当前大纲。

5. **生成全文**  
   大纲确认无误后，点击 **按此大纲生成全文** 或底部的 **🚀 生成全文**，系统将开始逐章撰写论文，并在任务列表中显示进度。

### 4. 下载论文

当任务状态变为 **✅ 已完成** 时，点击对应任务右侧的 **📥 领域分析**、**📥 文献综述** 等按钮，可分别下载各阶段文档（支持 `.md` 和 `.docx` 格式）。点击 **正式论文** 可下载最终全文。

### 5. 评价与建议

完成任务后，点击任务右侧的 **📝 评价** 按钮，可对本次生成结果打分（0~10分）并留下文字建议，帮助团队持续优化提示词和逻辑。

## ❓ 常见问题

**Q: 提示“请先设置 API 配置”怎么办？**  
A: 点击右上角配置按钮，正确填写 API Key 并测试通过即可。

**Q: 生成过程耗时很长？**  
A: 论文生成需多次调用 AI 接口，时间取决于章节数和字数要求，请耐心等待。您可以在任务列表中查看实时状态。

**Q: 如何重置用户身份？**  
A: 点击工具栏的 **重置用户** 按钮，系统会生成新的 `userId`，之后生成的任务将归属于新用户。旧任务仍可通过 **全部任务** 模式查看。

**Q: 如何查看所有人的任务？**  
A: 点击 **全部任务** 切换按钮（需管理员密钥登录），可看到所有用户的任务列表。

**Q: 生成的论文不够字数或质量不佳？**  
A: 尝试调整字数选择、补充额外要求或投喂更详细的参考材料。也可以修改大纲，使章节结构更合理。
